# 前言

最近学习压力很大，导致做什么事情都提不上性质。玩游戏的时候玩一会就累了，番剧也因为长篇追着太累很久没追了。正因如此，我迷上了 “有声漫画（アニメ動画）” 形式的视频。这些视频一般都在 10 到 20 多分钟内，使用静态的漫画和配音的视频形式讲述了许多有趣的故事。正因如此，在闲暇之余我会看一两篇有声漫画放松心情，有时也会故意不去看汉化字幕来练习日语听力。

但是目前为止，只有很少的有声漫画创作者创建了多语言频道，大多数质量很好的有声漫画创作者并没有成形的多语言账号运营。于是在广阔的互联网中，有许多优秀的 UP 主和译者默默地为喜爱有声漫画的人们翻译着有声漫画。然而，这些优秀的作品往往散落在不同的角落，难以被系统地发现和回顾。

在看到 B 站 UP 敔信 大佬收集的 B 站有声漫画汉化视频收集表格后，我收获了许多灵感。在收到大佬的授权许可后，使用 Astro 与 AI 辅助制作了这个网站：biliVManga。

在制作这个网站的过程，我也是第一次用 AI 辅助制作一个完整的 Astro 项目。在开发过程中我挑选了 Gemini 辅助设计，之前测试用 Gemini 生成过网页和样式，非常便利且生成的样式多样。不得不说，AI 确实大幅度减少了前端开发的许多时间，并且也减少了许多学习成本。但在开发过程中，也遇到了许多麻烦，这部分就在文章末尾再说吧。

# biliVManga

biliVManga 创建用于图形化展示 bilibili 大多数有声漫画汉化翻译视频并提供视频信息和索引。由发布平台 bilibili，以及有声漫画（Voice/Video Comic，日语「漫画」的罗马音 manga）的要素组成的名称。

目前已经部署在 bilivmanga.moeqy.com，通过读取敔信大佬的表格，在主页随机排序表格中所有有声漫画索引。

目前网站已经实现了：关键词搜索（标题、要素、原作者），本地收藏视频，读取表格，汉化 UP 主列表等功能。

  • 搜索支持输入关键词搜索翻译标题、原视频标题、原作者，但因为防止关键词冲突，不支持搜索汉化 UP 主。通过 “快速探索” 中列举的 tag 快捷搜索，可以帮助用户理解如何使用关键词；
    • 例如 UP 主 学姐 we 大佬翻译上传的视频太多了（足足 1000+），导致搜索 “学姐” 关键词时出现的都是该 UP 主的搜索结果，而不是与学姐元素有关的视频。所有 UP 主搜索功能由 “UP 主列表” 承担。
  • 支持通过本地浏览器缓存收藏视频，点击视频封面左上角的爱心按钮或者视频详情页的收藏按钮即可收藏视频。收藏列表中的视频会按照收藏顺序从新到旧排序。但清除浏览器缓存或者更换设备后收藏列表不会保存；
  • 支持点击首页右上角的 “导入表格” 按钮，通过本地浏览器缓存读取敔信大佬的表格数据。设计为仅在小站未更新最新表格数据时使用，导入文件后将会覆盖新的数据。但由于原表格不包含封面数据，导入后不会显示视频封面；
  • “UP 主列表” 将按照视频发布数量来排序（由多到少）参与汉化翻译视频的 B 站 UP 们。

点击主页的其中一个视频索引卡片，将会弹出一个详细信息窗口，里面的信息包括：

  • 原视频标题
  • 原视频链接
  • 原视频作者
  • 翻译视频链接
  • 译者（UID）
  • 发布日期

其中一些信息可能会因为表格数据未收录等原因不会显示出来，但大多数信息都可以通过汉化 UP 主提供的原视频链接获取。

在读取 B 站视频封面时，一开始是打算读取 API，但读取 API 获取视频封面的速度慢、读取次数太多，在占用网络资源的前提下还无法提供良好的浏览体验。于是我想了一个笨办法，通过脚本读取视频的 BV 号，随后将所有视频封面（压缩过后的）抓取到本地，最后生成一个带有视频封面路径的新表格。虽然刚开始在下载上很花费时间，但在之后更新会自动跳过已有的视频封面，且浏览时不用等待 API 相应，也算是比较折中的解决方案吧。

目前在 GitHub 上可以查看 biliVManga 所有页面的源代码。支持部署在本地使用（需要 Node.js 22 及以上版本），可以通过仓库页面的 readme.md 文件阅读详细部署流程以及使用声明。

# 后记

首先就是，非常感谢敔信大佬在听说我使用 ta 的表格制作索引网站时的大力支持和授权。大佬几乎把 B 站所有有声漫画转载视频记录和收集在他的表格里了，并且现在还在持续更新。

其次是我在使用 Gemini 辅助制作网站时的一些感想：AI 确实帮助了我很多，包括解决问题、查询知识、甚至一些代码层面的功能实现，AI 能够很好地完成我的指示。

但是，当项目涉及的功能越多（特别是代码长度超越 1000 行后），AI 逐渐开始达到限制并且逐渐开始生成出错，这在我还在制作中的博客年终总结项目中尤为体现。经常会出现不听从指令、原代码被覆盖的问题。并且，Gemini 的使用限额越来越少了，原本 Google AI Pro 能够几乎无限使用 Gemini 3 Pro 且不会触发任何上限，但现在几乎稍微用上一会就限制 Pro 模型的使用了。原本因为 Gemini 的多样性将 ChatGPT 的订阅取消了，但现在唯一能够支撑我订阅 Gemini 的原因可能就是附赠的 2TB 云盘吧（笑）。

总结就是，AI 确实方便了我写这些项目。使用 AI 我认为并不是一件比较可耻的事情，尽管我也在害怕我会过于依赖 AI，但如何命令 AI 以及展现自己的想法、项目的架构，依旧是项目负责人需要考虑且主导的重点。并且通过 AI 学习并判断新的信息也是当今重要的技术能力。

有声漫画虽然大多数都是比较轻松诙谐的恋爱短篇，但也有很多短小精悍的精品故事，学姐 we 大佬翻译并分类了很多这类作品，例如：

  • 【有声漫画】誓言就都留给时间，就请把从前留在今天
  • 【有声漫画】我遇到了坚定选择我的女孩…
  • 【有声漫画】十年植物人的青梅终于醒来，绝不放弃那小小的希望！
  • 【有声漫画】刚开始有多矜持，后面就有多主动
  • 【有声漫画】濒临离婚的我们因妻子事故失忆重回热恋，但她的失忆只是暂时
    • 这个不是大佬翻译的，但我个人比较喜欢。

如果感兴趣的话，希望您能去了解它们。
